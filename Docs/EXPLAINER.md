# 내가 한 것 — 쉬운 설명

---

## 문제 — 뭐가 부족했나

자율주행 차에는 레이더 센서가 달려 있다.
레이더는 앞에 있는 물체까지의 거리와, 그 물체가 **얼마나 빠르게 움직이는지(Doppler)** 를 측정한다.

AI가 "저게 차다, 저게 사람이다"를 잘 인식하려면 레이더 데이터가 많이 필요하다.
근데 실제로 차를 몰고 다니면서 데이터를 모으는 건 **비싸고 시간이 오래 걸린다.**

---

## 아이디어 — 가짜 데이터를 만들면 어떨까

> "카메라 사진으로 3D 장면을 복원해서, 거기서 레이더 데이터를 인공으로 뽑아내면 되지 않을까?"

실제로 찍은 사진들로 **3D Gaussian Splatting(3DGS)** 이라는 기술로 장면을 입체적으로 복원하고,
복원된 장면에서 레이더가 "쐈을 때" 돌아올 신호를 계산해서 **가짜 레이더 데이터**를 만든다.

```
실제 카메라 사진
      │
      ▼
3DGS로 3D 장면 복원 (PSNR 30.94 dB, 7.17M Gaussians)
      │
      ▼
가상의 새로운 시점에서 레이더 신호 계산
      │
      ▼
가짜 레이더 데이터 120장 생성
```

---

## 함정 — 그냥 만들면 안 됐다

처음엔 이렇게 만든 가짜 데이터로 학습했더니 오히려 성능이 나빠졌다.

원인을 파고들었더니 →
**가짜 데이터에서 물체의 속도값(Doppler)이 0으로 되어 있었다.**
마치 세상 모든 차가 멈춰있는 것처럼 학습된 것.

```
문제 상황:
  실제 레이더  → 차가 시속 60km로 달리고 있음  (Doppler = 16.7 m/s)
  가짜 레이더  → 차가 멈춰있음                  (Doppler = 0)

결과: AI가 "이 점이 움직이는 차인지 벽인지" 구분 못함 → 추적 실수 폭증
```

그래서 실제 nuScenes 데이터에서 GT(정답) 속도값을 가져와서
물리적으로 정확한 Doppler를 붙였다. 이게 `dynamic_compositor`다.

```
수정 후:
  가짜 레이더 포인트 위치  ← 3DGS depth에서 계산
  가짜 레이더 속도값       ← 실제 nuScenes GT velocity에서 복사 + 물리 변환
```

---

## 결과 — 뭐가 달라졌나

| | 실제 데이터만 | 가짜 데이터 섞기 |
|---|---|---|
| MOTA (전체 탐지 성능) | -0.048 | -0.054 |
| **같은 물체를 다른 물체로 착각한 횟수 (IDSW)** | **13번** | **3번** |
| PMBM 수학 알고리즘 | -0.177 | **3번** |

"저 차가 프레임마다 다른 차로 보이는 실수"가 **77% 줄었다.**

그리고 이 **3**이라는 숫자가 수십 년간 수학자들이 연구해서 만든
정교한 확률 추적 알고리즘(PMBM)과 **똑같은 수치**다.

---

## 왜 속도값(Doppler)이 그렇게 중요한가

레이더로 물체를 추적할 때 가장 어려운 건
"지난 프레임의 저 점이 이번 프레임의 이 점과 같은 물체인가?"를 판단하는 것이다.

- 위치만 보면: 여러 차가 비슷한 위치에 있을 때 헷갈린다
- **속도까지 보면**: "지난 프레임에서 시속 60km로 달리던 차"와 "이번 프레임에서 시속 60km로 달리는 차"를 연결할 수 있다

Doppler가 정확해야 AI가 "같은 물체"를 계속 같은 물체로 인식할 수 있다.

## 한 줄 요약

> 카메라로 만든 가상 세계에서 레이더 데이터를 뽑아내되,
> **속도값만 실제 데이터에서 빌려와 물리적으로 정확하게 만들었더니**
> AI가 물체를 놓치지 않고 일관되게 추적하는 능력이
> 수학 기반 전통 알고리즘 수준까지 올라왔다.
